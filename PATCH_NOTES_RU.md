# DonConnect Beta 3: патчноут

Версия: `0.13.0-beta.3.15`

## Hotfix 15 после Beta 3

- Таймерный виджет больше не показывает строку последнего донатера ни на первом запуске, ни до включения таймера в настройках.
- Стартовая раскладка алерта доната стала компактнее и ближе к центру, чтобы элементы не выглядели съехавшими при первом запуске.
- Очищены старые provider-значения из setup actions (`Client ID`, `Client Secret`, JWT, API keys, access tokens и custom endpoint), чтобы импортируемый `.sb` не содержал данные тестовой машины.
- Добавлен `BETA3_CLOUD_REFERENCE_RU.md` - подробный референс Beta 3 для переноса функций в облачную версию DonConnect.
- Пересобран `DonConnect.Beta3.sb` с кодом `0.13.0-beta.3.15`.

## Hotfix 14 после Beta 3

- Декоративная картинка таймера теперь хранится отдельно от декоративной картинки цели.
- Reset таймера больше не должен затрагивать картинки цели, а reset цели не трогает таймерный декор.
- Исправлены привязки слоя декора таймера в предпросмотре: перемещение, растягивание, поворот и сброс.
- Пересобран `DonConnect.Beta3.sb` с кодом `0.13.0-beta.3.14`.

## Hotfix 13 после Beta 3

- Логи донатов по дням теперь записываются строкой `dd/mm/yy - ник - сумма валюта - площадка - сообщение`.
- В OBS-доке кнопка «Открыть логи» находится рядом с «Обновить», чтобы стример быстрее находил папку логов.
- Вкладка «Док OBS» перенесена в конец редактора, после «Запрет».
- Растягивание слоёв алерта больше не должно сдвигать соседние элементы: текстовые части алерта переведены в независимые слои.
- Для цели исправлено растягивание полоски и текста за уголки, а сброс выбранных элементов возвращает их размеры и позиции по умолчанию.
- Для таймера исправлены слои площадок, блока таймера и подзаголовка; отдельный таймер больше не показывает последнего донатера.
- В алерт, таймер и лидерборд добавлены декоративные картинки.
## Hotfix 12 после Beta 3

- Исправлен баг предпросмотра, при котором растягивание одного элемента алерта могло сдвигать остальные элементы.
- Для алерта добавлены стабильные слоты раскладки: донатер, сумма, площадка, сообщение и медиа теперь ведут себя как независимые слои.
- Для общего direct editor добавлена компенсация reflow: при растягивании слоя в алерте, цели или таймере остальные редактируемые слои сохраняют визуальное положение.

## Hotfix 11 после Beta 3

- Логи донатов по дням теперь записываются в ту же папку DonConnect, которую открывает кнопка в редакторе и OBS-доке.
- Фон панели цели/таймера теперь можно растягивать по высоте прямо в предпросмотре.
- Перепроверены: прямое редактирование элементов алерта/цели/таймера, автозапуск локального сервера после перезапуска, экспорт/импорт профилей с медиа, отсутствие токенов в профилях, JS всех встроенных страниц и корректность импортируемого `.sb`.

## Hotfix 10 после Beta 3

- DonConnect теперь сам делает повторные попытки запуска локального сервера виджетов после старта Streamer.bot.
- Добавлен watchdog локального сервера: если сервер виджетов перестал отвечать, DonConnect пытается поднять его снова.
- Автосброс титров и лидерборда запускается при старте DonConnect и повторяется несколько минут, если Streamer.bot HTTP Server ещё не готов.
- В OBS-док добавлена кнопка открытия папки логов донатов.
- На вкладке «Док OBS» в редакторе кнопка логов теперь явно называется «Открыть папку логов по дням».

## Hotfix 9 после Beta 3

- Для старта стрима добавлен защищённый сессионный сброс: `DonConnect - Stream Start Reset`.
- Этот action сбрасывает титры и лидерборд максимум один раз за локальный день.
- Если стрим аварийно перезапустился в тот же день, повторный `Stream Online` не очищает данные второй раз.
- Сброс Streamer.bot Credits теперь строгий: DonConnect ждёт успешный ответ `/ClearCredits`; если HTTP Server Streamer.bot недоступен, сброс считается не выполненным.
- Автоочистка лидерборда при запуске тоже больше не срабатывает повторно в тот же день.

## Hotfix 8 после Beta 3

- В OBS-док добавлена кнопка «Сбросить титры».
- В пакет добавлен action `DonConnect - Reset Credits`; в Hotfix 9 он заменён на `DonConnect - Stream Start Reset`.
- Сброс титров очищает список DonConnect и пытается вызвать штатный HTTP-сброс Credits в Streamer.bot.
- После сброса DonConnect не возвращает старый файловый кэш титров сразу обратно.
- Перед упаковкой перепроверены таймер, цель, шрифты, профили, логи донатов, автозапуск и отсутствие встроенных секретов DonationAlerts.

## Beta 3

- Добавлен отдельный таймер сбора в виджет цели: можно включить его отдельно от донатного таймера, указать дату/время окончания, менять шрифт, положение, размер, поворот и порядок слоя.
- Фильтрация титров Streamer.bot стала безопаснее: если DonConnect не узнаёт схему `settings.json`, он оставляет секции титров как есть, а не скрывает их из-за отсутствующих массивов настроек.
- Удалены встроенные данные приложения DonationAlerts: теперь пользователь создаёт своё приложение DA и вставляет Client ID + Client Secret на странице подключения площадок.
- Автозапуск разделён на два шага: даже если провайдеры не стартовали из-за токена, локальный сервер редактора и OBS-виджетов всё равно пытается подняться.
- Страница подключения DonationAlerts стала понятнее: есть кнопка создания приложения, подсказка с Redirect URL и проверка, что ключи заполнены до авторизации.
- Профили редактора теперь явно сохраняют настройки виджетов и используемые медиа, но не токены площадок.
- Streamlabs, Donate.Stream, deStream, DonatePay, DonateX, StreamElements, ODA и Generic API оставлены как существующие провайдеры; StreamEngine пока остаётся плейсхолдером «В разработке» до стабильного API.
- Файл импорта для релиза: `DonConnect.Beta3.sb`.

## Главное

Beta 2 превращает DonConnect из простого моста донатов в полноценный локальный набор виджетов для Streamer.bot и OBS. Теперь стример может открыть один браузерный редактор, настроить внешний вид донат-алерта, цели, таймера, титров, лидерборда и OBS-дока, а затем сразу скопировать нужные ссылки в OBS.

Редактор запускается самим расширением Streamer.bot. Ничего дополнительно устанавливать не нужно.

## Что добавлено

- Встроенный браузерный редактор DonConnect на `http://127.0.0.1:3987/donconnect/editor`.
- Один главный action для пользователя: `DonConnect - Widget Editor`, который запускает сервер и открывает редактор.
- OBS Browser Source URL для донат-алерта, цели, таймера, титров и лидерборда.
- OBS-док `http://127.0.0.1:3987/donconnect/dock` для просмотра последних донатов и повтора алертов.
- Донат-алерт с настройками текста, цветов, шрифтов, расположения медиа, времени показа и тестовыми оповещениями.
- Медиатека для алертов: изображения, GIF, видео и звуки.
- Повтор последних донатов без повторного влияния на цель, таймер и титры.
- Озвучка текста доната через доступные голоса Windows.
- Цель с пресетами, настройкой полосы, вертикальным режимом, инверсией, заполнением картинкой и отдельными настройками текста.
- Таймер с пресетами, обычным отсчетом, отсчетом вперед и конвертацией доната во время.
- Титры с интеграцией данных Streamer.bot Credits, паузой, отключением секций и живой настройкой внешнего вида.
- Лидерборд с пресетами, ограничением строк, редактированием имен и удалением тестовых/лишних записей.
- Раздел запретов для ников и слов, которые не должны попадать в виджеты.
- Переключение языка редактора: английский, русский, украинский.
- Поддержка локальных шрифтов Windows и выбор шрифтов по разделам виджета.
- Настройки сохраняются в JSON-файлы в папке `DonConnect` рядом со Streamer.bot, а не в AppData.
- Проверка новых версий при запуске через `version.json` на GitHub.
- Чистый релизный файл импорта `DonConnect.Beta3.sb` вместо старых `DonConnect.install*.sb`.

## Подключение DonationAlerts

В Beta 3 встроенные данные приложения DonationAlerts удалены. Пользователь должен создать своё приложение DonationAlerts и вставить ключи в браузерной странице подключения.

Достаточно:

1. Импортировать `DonConnect.Beta3.sb`.
2. Запустить `DonConnect - Подключение площадок`.
3. В строке DonationAlerts нажать **Создать приложение**.
4. В DonationAlerts указать Redirect URL `http://127.0.0.1:8597/donconnect/donationalerts/callback/`.
5. Вставить Client ID и Client Secret в DonConnect.
6. Нажать **Сохранить подключения** и **Авторизовать DonationAlerts**.

## Важно про дубли донатов

Если донаты приходят через агрегатор и через исходную площадку одновременно, возможны дубли. Например, StreamElements может отправить донат через DonatePay EU, а DonatePay EU тоже может быть подключен напрямую. ODA также может собирать несколько площадок внутри себя.

В DonConnect есть дедупликация, но разные API иногда отдают разные id, ник или текст операции. Перед живым стримом стоит проверить подключенные площадки тестовыми донатами.

## Что проверить тестерам

- Импортируется ли `DonConnect.Beta3.sb` без старых лишних actions.
- Открывается ли `DonConnect - Widget Editor`.
- Работают ли URL в OBS Browser Source.
- Заполняется ли цель от тестового и реального доната.
- Продлевается ли таймер от тестового и реального доната.
- Показывается ли донат-алерт и работает ли повтор из OBS-дока.
- Корректно ли работают DonationAlerts, DonatePay, DonateX, StreamElements, ODA и другие подключенные провайдеры.
- Не появляются ли дубли при одновременном подключении агрегатора и исходной площадки.

## Известные ограничения Beta 3

- Это бета-версия, перед живым стримом лучше проверить все виджеты на тестовой сцене OBS.
- Проверка новых версий работает только в версиях, где уже был добавлен механизм update-check. Очень старые сборки могут не показать уведомление о Beta 3.
- Некоторые площадки могут менять API или отдавать неполные данные о донате, поэтому интеграции провайдеров всё ещё требуют живого тестирования.
